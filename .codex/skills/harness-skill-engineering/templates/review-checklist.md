# Skill 上线检查清单

## 结构

- [ ] 目录名是 kebab-case。
- [ ] 目录名不超过 64 个字符。
- [ ] 主文件名是全大写 `SKILL.md`。
- [ ] `SKILL.md` 没有写成百科全书。
- [ ] 长规范进入 `reference/`。
- [ ] 稳定格式进入 `templates/`。
- [ ] 长示例进入 `examples/`。
- [ ] 确定性逻辑进入 `scripts/`。

## 触发

- [ ] `description` 说明了做什么。
- [ ] `description` 覆盖用户真实说法和同义表达。
- [ ] `description` 有不适用边界。
- [ ] 没有过泛词导致误触发。
- [ ] 没有超过预算。
- [ ] 准备了应触发和不应触发用例。

## 权限

- [ ] 权限符合最小权限。
- [ ] 有副作用任务设置 `disable-model-invocation: true`。
- [ ] 没有 `Bash(*)`。
- [ ] Bash 命令使用具体前缀白名单。
- [ ] 动态命令只收集上下文，不执行高风险动作。
- [ ] 输出会提醒用户确认高风险影响范围。

## 渐进式加载

- [ ] 主文件有 Quick Reference。
- [ ] 每个辅助文件引用都有“何时加载、去哪加载、获得什么”。
- [ ] 低频知识不会在每次触发时全量加载。
- [ ] 模板、脚本、参考文档职责清晰。

## 验证

- [ ] 通过触发测试。
- [ ] 通过功能测试。
- [ ] 通过安全检查。
- [ ] 与无 Skill 情况相比减少纠正次数。
- [ ] 用户反复纠正点已回写到 Skill。
